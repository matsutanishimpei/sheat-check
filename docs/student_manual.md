# 🎓 Seats & Check - 学生向け利用マニュアル

本システムは、講義中の理解度や要望（「難しい」「声が遠い」など）を、リアルタイムに教員へ匿名で伝えることができるシステムです。

---

## 📱 1. 講義へのチェックイン方法
授業が始まると、教室のプロジェクター等に **「学生用チェックイン QR」** が表示されます。

1. スマートフォンのカメラまたは QR コードリーダーで、画面に表示された QR コードをスキャンします。
2. 表示されたブラウザでアプリを開きます（推奨ブラウザ: Safari, Google Chrome 等の最新版）。
3. QR コードの代わりに、指定された URL（例: `https://seat-check.pages.dev/student/[ROOM_ID]`）へ直接アクセスすることでもチェックインできます。

---

## 🪑 2. 自分の座席の選択
チェックインに成功すると、現在の教室の座席図（レイアウト）が表示されます。

1. **座席図から、自分が現在座っている席をタップ** して選択します。
   * 選択すると、席のマークが黄色（またはアクティブ色）に変わります。
   * 既に他の学生が選択している席は選べません。
2. 席を選択したら、画面下部の **「この席で確定」** ボタンをタップします。
3. 席の変更を行いたい場合は、一度確定した後でも画面下部の **「席の変更 (Change Seat)」** ボタンから再度選び直すことができます。

---

## 💬 3. 講義中のフィードバック（理解度・要望）の送信
席が確定すると、フィードバック送信画面に移行します。
講義中、ご自身の状況や教員への要望に合わせて、以下の6つのボタンをいつでもタップして送信できます。

| ボタン | 送信されるステータス・意味 |
| :--- | :--- |
| 🟢 **バッチリ！** | 講義の内容を非常によく理解できています。 |
| 🟢 **なるほど！** | 解説に納得、または新しい気づきがありました。 |
| 🔴 **メモ待って** | 板書やスライドの進行が速く、ノートを取る時間がほしいです。 |
| 🔴 **むずかしい** | 現在の解説やロジックが難しく、ついていけていません。 |
| 🔴 **声が遠い** | マイクの音量や教員の話し声が小さく、聞き取りにくいです。 |
| ⚪ **暑い・寒い** | 教室の空調や温度（暑さ・寒さ）を調整してほしいです。 |

* **匿名性について**: 
  送信したフィードバックやコメントは、教員側からは「どの席から送信されたか」のみが視覚的にわかります。学生の個人名（学籍番号や氏名など）は教員画面には表示されないため、安心して率直な理解度を送信してください。

---

## 📳 4. 通信トラブル（リアルタイム接続切れ）時の動作
通信環境が悪化し、リアルタイムでの状態同期が切断された場合、アプリが自動的に **「HTTPバックアップ同期モード（7秒間隔の自動再送）」** に切り替わります。
* 画面上部に自動的に警告が表示されますが、そのままフィードバックボタンをタップし続ければ、7秒間隔でバックグラウンドから教員側へ送信が維持されます。手動でリロード（再読み込み）する必要はありません。
